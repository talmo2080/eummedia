# eumwiki 운영 규칙 (Karpathy LLM Wiki 패턴)

이 폴더는 **원본 자료를 던져두면(raw) → 클론(Claude)이 정리 페이지를 생성·연결(wiki)** 하는 개인 지식 위키다.
핵심 원칙: **노트 작성·정리·연결·목록 갱신은 전부 클론이 한다. 사용자는 raw에 자료만 넣고 질문만 한다.**

## 폴더 구조

```
eumwiki/
├── CLAUDE.md          ← 이 규칙 파일
├── README.md          ← 사람용 안내
├── _템플릿.md          ← 페이지 작성 템플릿
├── raw/               ← 사용자가 원본 자료를 그냥 던져 넣는 곳 (플랫, 하위폴더 없음)
└── wiki/              ← 클론이 생성하는 정리 페이지
    ├── index.md       ← 전체 목차 (항상 최신 유지)
    ├── log.md         ← 작업 이력
    ├── 01-두피관리-미용/
    ├── 02-미디어-편집-콘텐츠/
    ├── 03-개발-툴-자동화/
    ├── 04-사업-기획-마케팅/
    └── 05-개인학습-아이디어/
```

- **raw/** : 기사·PDF·메모·대본·책노트 등 원본. 사용자가 자유롭게 던져 넣는다. 클론은 이 폴더의 파일을 **읽기만** 하고 수정하지 않는다.
- **wiki/** : 클론이 만드는 결과물. 위 5개 분류를 사용한다.

## 트리거: "위키에 반영해줘" (raw → wiki 반영)

사용자가 raw/에 파일을 넣고 반영을 요청하면 **아래 순서를 한 번에** 수행한다.

1. **읽기** — 지정된(또는 새로 들어온) raw 파일을 읽는다. 특정 파일을 지목하지 않으면, `wiki/log.md`에 아직 반영 기록이 없는 raw 파일을 대상으로 한다.
2. **분류·판단** — 내용이 5개 분류 중 어디에 속하는지 정하고, **관련 기존 페이지가 있는지 먼저 찾는다.**
3. **생성 또는 보강** — ⚠️ 새 페이지를 무조건 만들지 않는다. **관련 기존 페이지가 있으면 보강을 우선**하고, 없을 때만 새 페이지를 만든다.
4. **교차링크** — 새/수정 페이지와 관련 페이지를 상대경로 링크로 서로 연결한다. (`[제목](../03-개발-툴-자동화/노트.md)`)
5. **index 갱신** — `wiki/index.md`의 목차에 추가/변경 사항을 반영한다.
6. **log 기록** — `wiki/log.md`에 `날짜 · 반영한 raw 파일 · 생성/보강한 페이지 · 요약` 한 줄을 추가한다.
7. **보고** — 무엇을 새로 만들고 무엇을 보강했는지, 어떤 링크를 걸었는지 사용자에게 요약해 알린다.

원본의 출처를 각 wiki 페이지 상단에 남긴다: `> 출처: raw/파일명`

## 트리거: "건강검진" / "lint" (위키 점검)

위키 전체를 점검하고 개선한다.

- **인덱스 정합성** — index.md에 없는 페이지 / index.md에만 있고 실제 없는 페이지
- **교차참조 누락** — 서로 연결되어야 하는데 링크가 빠진 페이지, 깨진 링크
- **중복·모순** — 같은 내용이 여러 곳에 흩어졌거나 서로 어긋나는 서술 → 병합/정리 제안
- **오래된 정보** — 갱신이 필요한 내용 표시
- **갭(gap)** — 추가로 조사·정리하면 좋은 빠진 주제 제안

발견 → 가능한 것은 바로 고치고, 판단이 필요한 것은 목록으로 보고한다. 점검 결과도 log.md에 남긴다.

## 페이지 작성 원칙

- 한 페이지 = 한 주제. 파일명은 짧고 명확하게 (`두피-지성-관리루틴.md`).
- 상단에 날짜·분류·태그·출처, 본문은 [`_템플릿.md`](_템플릿.md) 구조를 따른다.
- 원본을 그대로 붙여넣지 말고 **핵심을 재구성·요약**한다. 인용이 필요하면 명시한다.
- 확신이 낮은 내용은 추정임을 표시한다. 사실 왜곡 금지.
