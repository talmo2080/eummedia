## 🔄 작업 중 체크리스트

### 매 작업 사이클 순서 — 반드시

1. 클론 분석 보고
2. 옵스 검증
3. 세연님 OK
4. 클론 코드 적용
5. dev 검증 (세연님 직접 또는 클론 캡처)
6. 세연님 최종 OK
7. commit + push

### 작업 중 절대 원칙

- [ ] 옵스 검증 없이 코드 적용 금지
- [ ] 세연님 OK 없이 commit 금지
- [ ] 다른 항목 미리 손대기 금지
- [ ] 수정 파일 지시된 파일 외 0건 유지
- [ ] 광고 코드 손대지 않음
- [ ] 빨간 깃발 발생 시 즉시 멈춤 + 보고

### 빨간 깃발 판단 기준

- 지시 외 영역에서 변경 발견
- 동일 메시지 두 번 이상
- 세연님이 "이상해" 한 마디
- modified 파일 지시 외 1건이라도 발생

## 📌 메타인지 박제 누적 (협업 중 발견된 원칙)

1. **modified 파일은 첫 보고 1순위 빨간 깃발** — 출처와 검증 이력 확인이 먼저
2. **동일 메시지 두 번이면 추측 금지, 멈춰 보고**
3. **부수 발견은 옵션 균형 제시 + 세연님 결정** — 임의 처리 금지
4. **선의로 미리 손대기 = 금지** — "제가 이건 먼저 해두면 도움 될까요?" 후 OK 받기
5. **(5/13)** 옵스가 박제 본다고 만능 아님. **클론의 빨간 깃발이 진짜 보호막**
6. **(5/13)** 법적 1차 자료는 이미지 자체 보관 + 의심 시 재확인
7. **(5/14 오전)** 옵스가 ✅ 통과시킨 부수 발견이라도 라인별 정밀 확인으로 한 번 더 보면 사고를 막을 수 있음
8. **(5/14 오후)** 세연님이 "이상해" 한 마디 하면 옵스 추측 금지, 정확히 옵션으로 분해해서 묻기
9. **(5/14 긴급)** 시스템 변경 시 그 시스템 참조하는 모든 영역 한 번 더 점검 — Home.jsx 채널 카드 누락 사고가 그 예
10. **(5/14 신규)** 옵스도 클론도 새 세션 = 백지. 박제 없으면 아무것도 모름 — 박제 파일 4개 첨부가 협업의 다리
11. **(5/15)** 회색 박스(스켈레톤) 1초 이상 영구 표시 = fetch 영구 실패 신호. 검증 시 즉시 빨간 깃발.
12. **(5/15)** 한국어 요청의 미세 동사 차이는 옵스 추측 금지 — 교체/이동/추가/제거 옵션 균형으로 재확인 필수. 5/15 옵스 실수 3건 누적에서 도출.
13. **(5/15)** 어제 발견 → 오늘 해소 가능 — 박제 시스템의 시간차 정확성. P-01 채널 카드 사고 (5/14 발견 → 5/15 해소)가 예.
14. **(5/15)** F12 콘솔 빨간 에러는 출처 분리가 먼저. 사이트 코드 vs 브라우저 확장 — 에러 객체 shape 정밀 분석 (name, code 타입, httpStatus 속성명 표준 여부)으로 결정. 시크릿 모드 1회 캡처가 1차 분리 기준.
15. **(5/15)** 옵스는 작은 영역 의논 전에 큰 그림(수익 모델 + 매체 정체성) 점검을 자주 해야 함. 세연님 본능이 본질을 되돌려주기 전에 옵스가 먼저 확인. 5/15 헤더 디자인 v3~v10 시간 낭비 + 광고 시스템 누락이 그 예.
16. **(5/16)** 옵스가 세연님 정체성을 잘못 알고 박제 영역 광범위. "27년 매거진"은 사실 두피업 27년 (닥터리부트, 두피전문가). 매거진은 이제 시작. "웹개발 도전 중"은 사실 웹개발자. 교훈: 옵스가 인물 정체성을 추측하면 안 됨. 세연님 직접 확인 필수. 박제 시 사실 정합 검증 영역. 25번째 commit 후미 메시지에 잘못된 사실 박힘 → 26번째 commit으로 정정.
17. **(5/16)** 옵스가 5/12 이전 작업 영역 미인지 함정 (Phase 1 / 5/8 작업 사실 미인지). 옵스 "신규 페이지 작업" 지시 → 실제 `Advertise.jsx` 337줄 이미 존재. 클론 자가 빨간 깃발 + 점검 보고로 함정 해소. 교훈: 옵스가 신규 작업 지시 전 기존 코드 영역 점검 필수. 클론은 신규 작업 명시 시 즉시 grep/glob으로 기존 영역 확인 → 발견 시 멈춰 보고. view 영역 명령(점검 → 보강 모드 전환) 자연 진입 패턴 박제.
18. **(5/17)** 한글 음절 시각 유사 영역 정수 — 등록번호 "이"·"아" 정정 사례 (5/17) + 5/13 등록증 OCR "상"·"아" 정정 사례 영역 정합. 등록증·법적 문서 영역 1자 차이 영역 정밀 영역. 시각 유사 한글 음절 (아·이·상·어·오·우 등) 영역 grep 전체 검색 + 사실 정합 검증 필수.
19. **(5/17)** 과거 박제 사실 기록 영역은 정정 영역 아님 — "주간인터넷신문 → 인터넷신문" 박제 메모 영역은 5/12 정정 사실 기록. 박제 정정 영역 = 사실 기록 변형 영역 + 메타인지 #3 영역 충돌. 일괄 정정 지시 받으면 실제 사용 영역(코드/카피) vs 박제 기록 영역(docs) 분리 grep 보고 → 옵스 결정.
20. **(5/17)** Hero 안 박스 디자인 영역 = 그라데이션 영역 정합 — 흰 박스(보수) vs 반투명(통일) 영역. Hero 안 박스는 반투명 디자인 영역 자연 (Advertise.jsx Hero 안 광고 철학 박스 패턴 영역). Subscribe.jsx 회사정보 박스 영역 1차 적용 흰 박스 → 2차 정정 반투명 영역 정합.
21. **(5/17)** 부수 발견 영역 분리 영역 — "토요일 일괄 정정" 명시 영역 = BENEFITS + STEPS 한정. Hero 카피 잔여 = 부수 발견 영역 + 별도 사이클 영역 (옵스 결정 영역). 일괄 명시 영역 + 명시 본문 한정 정정 + 추가 영역 보고 = 메타인지 #3 영역 정합.
22. **(5/17)** 등록증 매핑 영역 vs 시각 감성 카피 영역 분리 — About.jsx CHANNELS desc = 등록증 정합 영역. ChannelList.jsx CHANNEL_META desc = 5/14 박제 감성 카피 영역. 두 영역 의미 영역 분리 + 일괄 정정 영역 X. 옵스 "고양·일산" / "라이프스타일·소비" 잔여 영역 추정 함정 = 시각 카피 영역 추가 정정 영역 X (메타인지 #9 영역 정합).
