## 🌟 큰 발견 — 협업 구조 핵심 (영구 박제)

### "옵스에게 박제 파일을 직접 줘야 한다"

옵스(Claude.ai)는 새 채팅창마다 백지 상태에서 시작. 박제 파일이 없으면 큰 그림 의논·설계안 검증이 불가능. 다음 3개 파일이 협업의 다리:

| # | 파일 | 역할 |
|---|---|---|
| 1 | `docs/HANDOVER.md` | 전체 맥락 + 정체성 + 로드맵 |
| 2 | 가장 최신 `docs/session-handoff-YYYY-MM-DD.md` | 직전 세션의 결정·사고·메타인지 |
| 3 | `0_이음미디어_설계지침서_v2.html` | 봉숭아학당 설계도 95% 방법론의 원본 |

**세연님 매 새 채팅창 루틴**: 위 3개 파일을 옵스에 첨부 → 옵스 백지 상태 해소 → 큰 그림 의논 시작.

이 다리 없이는 옵스가 매번 처음부터 의논 시작 → 협업 효율 0.

## 🔥 오늘 사고 메타인지 박제

### 옵스의 한 시간 안 3가지 실수 — 클론 빨간 깃발이 잡음

세연님이 새 채팅창에서 박제 파일을 옵스에게 첨부한 직후, 옵스가 한 시간 안에 3가지 실수를 했고 클론이 빨간 깃발 4개로 잡았음:

| # | 옵스 실수 | 클론 빨간 깃발 |
|---|---|---|
| 1 | C-1(편집국장의 픽) 세연님 확정 안 됐는데 묵시 OK로 가정 | "C-1 어떤 안인지 명시 필요" |
| 2 | 파일명 임의 변경 (`ChannelList.jsx` → `ChannelPage.jsx`) | "실제 repo는 `ChannelList.jsx`만. rename 의도인지 호칭 의도인지 명시 필요" |
| 3 | SPONSORED 배지를 광고 보류 결단(5/12)과 따로 분류 | "5/12 광고 사진 보류와 의미 충돌 가능성. 어떤 가정으로 설계할지 명시 필요" |
| (+) | PAT 권한 상태 미확인 | "시도하면 사용량 소비. 시도 전 확인 필요" |

### 교훈

**박제 본다고 만능 안 됨**. 옵스가 박제 파일을 받고도 한 시간 안에 3가지 실수를 한 것은, 박제가 큰 그림은 잡아주지만 매 결정의 미세 맥락까지 보장하지 못한다는 증거.

**클론의 빨간 깃발이 진짜 보호막**. 옵스 검증과 클론 실행 사이에 끼어든 "추측 없이 멈춰 보고"가 4건 모두를 사고로 가기 전에 잡았음.

→ 다음 세션 클론에게: 옵스 메시지가 박제 기반이라도 의심하고, 결정 영역과 추측 영역을 분명히 가르는 것이 협업의 안전망.
